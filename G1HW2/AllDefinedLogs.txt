공용 메시지 로그(data server, cache server, client)
- FLAG:1 (설명: 데이터 서버의 파일 준비가 끝나면 다른 cache와 client에게 시작하라고 보내는 신호, 0은 종료 신호)
- FILE:file_num:file_data:max_file_num:request_cnt (설명 파일 데이터 전송 시 , FILE: 파일 번호 : 파일 데이터:캐시에 보내지는 파일 중 가장 크기가 큰 파일 번호:파일 요청 횟수)
- REQUEST:file_num (설명: 파일 요청 메시지 , REQUEST: 파일 번호)

데이터 서버와 클라이언트 간 정의한 로그
- RANDOM:client에서 data server에 요청할 random 1000개 파일 목록
    ㄴRANDOM:random_list 클라이언트에서 data server에 파일 목록을 줄 때 형식

캐시 서버와 클라이언트 간 로그
- Cache Hit
- Cache Miss

캐시 서버
① "데이터 서버 {host}:{port}에 연결되었습니다." - 데이터 서버와 연결 성공
② "데이터 서버에 연결 중 오류 발생: {e}"
③ "파일 {file_num}의 request_cnt 감소: {request_cnt}"
    - 파일을 클라이언트에게 전송하여 파일 요청횟수가 -1 되었을 때 출력
④ "파일 {file_num}의 request_cnt가 0이 되어 캐시에서 제거되었습니다. 현재 캐시 사용량: {cache_size} KB"
    - 캐시에 저장된 파일의 request_cnt(요청 횟수)가 0이 되어 캐시에서 삭제
⑤ "FLAG 1을 수신했습니다."
⑥ "cache_size: {cache_size}, 데이터 서버로 부터 받은 파일: {file_num}, 남은 공간: {free_space}"
    - 데이터 서버로부터 파일을 받아서 캐시에 저장 후, 정보 출력
⑦ "데이터 서버에 {file_num}번 파일 요청 전송"
⑧ "파일 {file_num}을(를) 캐시에 저장했습니다. 현재 캐시 사용량: {cache_size} KB"
⑨ "캐시 용량 부족으로 파일 {file_num}을(를) 캐시에 저장하지 못했습니다."
⑩ "연결된 클라이언트: {addr}" - 클라이언트가 캐시 서버에 연결됐을 때 출력
⑪ "클라이언트로부터 파일 {file_num} 요청 수신" 
⑫ "Cache Hit: {file_num}번 파일 캐시에서 {CACHE_TO_CLIENT_SPEED}로 전송"
⑬ "Cache Miss: {file_num}번 파일 캐시에 없음, 데이터 서버로 요청"
⑭ "클라이언트 연결 종료: {addr}"
⑮ "데이터 서버와의 연결이 종료되었습니다."
